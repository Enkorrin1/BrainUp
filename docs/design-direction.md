# Дизайн-направление

## Главный принцип

BrainUp не должен быть прямой копией текущего `D:\Project\LogicLike`. Из исходного проекта можно брать инженерные решения, учебную структуру и опыт работы с логическими задачами, но визуальный дизайн, экранные композиции, персонажи, награды, карта и тон интерфейса должны быть оригинальными.

## Что можно переиспользовать

- Flutter/Dart стек.
- Архитектурное разделение `app`, `data`, `domain`, `features`, `l10n`, `theme`.
- Идею коротких логических задач.
- Доменные сущности уроков, шагов, пазлов, попыток и прогресса.
- Подход к локализации EN/RU.
- Подход к тестированию.

## Что нельзя копировать

- Текущую главную композицию `LogicLike`.
- Текущие карточки курсов один в один.
- Текущую палитру как основной визуальный стиль.
- Текущих персонажей и иллюстрации как бренд-ядро.
- Текущие reward cards и collection cards без переработки.
- Тексты, которые звучат как прямой перенос.
- Exact UI паттерны, если они делают новый продукт визуально тем же самым.

## Новый визуальный образ

Рекомендуемое направление: “логическое путешествие”.

Продукт может ощущаться как интерактивная карта островов, лабораторий, планет или мастерских мышления, где каждый узел - короткая тренировка навыка.

Возможные темы:

- острова логики;
- космический маршрут знаний;
- город головоломок;
- мастерская изобретателя;
- экспедиция по мирам мышления.

Для первого прототипа лучше выбрать одну тему и не смешивать все сразу.

## Рекомендованная тема для прототипа

Рекомендация: “космический маршрут знаний”.

Почему:

- хорошо подходит к карте прогресса;
- легко показать locked/current/completed узлы;
- можно сделать оригинальные планеты, станции, орбиты и награды;
- подходит детям 4-8;
- не требует копировать текущий LogicLike hub;
- естественно поддерживает Duolingo-style progression.

## Визуальные правила

- Первый экран - карта, не каталог.
- Карта должна быть узнаваемым главным объектом продукта.
- Узлы должны отличаться состояниями: locked, current, completed, review, reward.
- Интерфейс должен быть детским, но не хаотичным.
- Цвета должны быть яркими, но не превращаться в одну монотонную палитру.
- Награды должны выглядеть как часть выбранной темы.
- Родительский раздел должен быть спокойнее и взрослее child flow.

## Отличия от LogicLike

| Область | LogicLike источник | BrainUp прототип |
| --- | --- | --- |
| Главный экран | learning hub / курсы / миссия | Duolingo-style карта |
| Навигация | карточки разделов | последовательные path nodes |
| Визуальный образ | текущая семейная edtech стилистика | новая тема, например космический маршрут |
| Награды | коллекция/стикеры | XP, streak, hearts, тематические unlocks |
| Урок | puzzle flow | короткий node lesson с result moment |
| Ошибки | статистика/подсказки | mistake review как отдельный loop |

## UI-компоненты прототипа

Нужно спроектировать заново:

- `PathScreen`;
- path node;
- current lesson CTA;
- locked node;
- completed node;
- review node;
- lesson header;
- answer option card;
- result screen;
- XP/streak/hearts widgets;
- mistake review entry;
- parent entry.

Можно временно оставить существующие puzzle renderers, но их контейнер, header, feedback и result flow должны выглядеть по-новому.

## Copy и тон

Тон должен быть:

- короткий;
- дружелюбный;
- уверенный;
- без давления;
- понятный ребенку;
- спокойный для родителя.

Не использовать длинные объяснения на главном экране. Главный экран должен вести действием, а не инструкцией.

## Definition of Unique Design

Дизайн прототипа считается достаточно уникальным, если:

- по скриншоту первого экрана его нельзя спутать с текущим `LogicLike`;
- карта является главным визуальным паттерном;
- палитра, узлы, награды и композиция отличаются от исходного проекта;
- персонажи и иллюстрации не являются прямым переносом;
- child flow и parent flow имеют собственные визуальные роли;
- UI поддерживает Duolingo-style привычку, а не LogicLike hub-first опыт.
