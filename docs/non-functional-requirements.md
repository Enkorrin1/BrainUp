# Нефункциональные требования

## Производительность

- Первый экран должен открываться быстро даже на слабом устройстве.
- Переход между заданиями должен ощущаться мгновенным.
- Проверка ответа должна происходить без заметной задержки.

## Надежность

- Прогресс нельзя терять после закрытия приложения.
- Повторная отправка ответа не должна ломать результат урока.
- При ошибке сети пользователь должен видеть понятное состояние.

## Доступность

- Основные элементы должны быть крупными и удобными для ребенка.
- Цвет не должен быть единственным способом понять статус.
- Текст должен быть читаемым.
- Интерактивные элементы должны иметь достаточно большие зоны нажатия.

## Безопасность

- Не хранить лишние персональные данные ребенка.
- Не показывать публичные профили детей в MVP.
- Родительские и административные функции должны быть защищены авторизацией.

## Масштабируемость

- Новые типы заданий должны добавляться без переписывания всего урока.
- Контент должен быть отделен от логики приложения.
- Модель навыков должна поддерживать prerequisites и уровни сложности.

## Локализация

- Тексты интерфейса должны быть вынесены так, чтобы позже можно было добавить другие языки.
- Задания должны хранить язык контента явно или через структуру курса.
