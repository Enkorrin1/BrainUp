# Roadmap

## Phase 0. Product Foundation

Цель: зафиксировать продуктовую основу и подготовить новый проект к разработке на базе `D:\Project\LogicLike`.

Результаты:

- финальная цель продукта;
- анализ исходного LogicLike;
- product spec;
- Flutter/Dart tech stack;
- модель контента;
- UX-цикл обучения;
- MVP scope;
- критерии готовности.

## Phase 1. Prototype

Цель: собрать кликабельный Flutter-прототип главного Duolingo-style цикла.

Функции:

- карта обучения;
- экран урока;
- проверка ответов;
- экран результата;
- XP и streak;
- hearts;
- моковое повторение ошибки;
- несколько задач из текущей LogicLike-модели.

Критерий завершения:

- пользователь может пройти один урок от начала до конца;
- прогресс визуально обновляется на карте;
- UX понятен без инструкции;
- все видимые строки идут через l10n.

## Phase 2. MVP

Цель: выпустить первую рабочую Flutter-версию с реальным оригинальным логическим контентом.

Функции:

- 3-5 навыков;
- 20-30 уроков;
- 100-150 задач;
- сохранение прогресса;
- повторение ошибок;
- daily goal;
- streak;
- hearts;
- базовая родительская аналитика;
- локализация EN/RU.

Критерий завершения:

- ребенок может заниматься несколько дней подряд;
- система хранит прогресс локально;
- есть достаточно контента для первой недели занятий;
- `flutter analyze` проходит;
- `flutter test` проходит;
- выполнен Android emulator smoke test;
- подготовлен iOS handoff checklist для macOS/Xcode.

## Phase 3. Retention and Learning Quality

Цель: улучшить возвращаемость и образовательный эффект.

Функции:

- диагностика уровня;
- adaptive review;
- умное повторение;
- улучшенные объяснения;
- достижения;
- недельные цели;
- родительский отчет;
- рекомендации по слабым навыкам.

Критерий завершения:

- продукт показывает рост возвращаемости;
- точность ребенка по слабым навыкам улучшается после повторений.

## Phase 4. Content Scale

Цель: масштабировать библиотеку заданий и тем.

Функции:

- удобный внутренний формат контента;
- новые типы пазлов;
- больше возрастных треков;
- проверка качества заданий;
- теги навыков и сложности;
- импорт/экспорт контента;
- подготовка к редактору контента.

Критерий завершения:

- новые уроки можно добавлять без переписывания логики приложения.

## Phase 5. Cloud, Accounts, Monetization

Цель: превратить MVP в устойчивый продукт.

Функции:

- родительский аккаунт;
- облачная синхронизация;
- несколько детских профилей;
- подписка;
- платежи;
- расширенная аналитика;
- кабинет преподавателя в перспективе.

Критерий завершения:

- продукт поддерживает платные сценарии;
- родители видят понятную ценность;
- прогресс не зависит от одного устройства.
