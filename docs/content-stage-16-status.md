# Content Stage 16 Status

## Goal

Define BrainUp helper characters so puzzle feedback, hints, rewards, and worlds
feel warmer and more consistent.

## Delivered

- Added `docs/content-characters.md`.
- Defined 6 core helper characters:
  - Brainy: main guide;
  - Lumi: memory guide;
  - Quadra: shape and space guide;
  - Numba: math guide;
  - Rulo: logic guide;
  - Mira: attention guide.
- For every character, documented:
  - stable `characterId`;
  - role and target skills;
  - visual silhouette;
  - personality and voice;
  - preferred worlds;
  - preferred puzzle families;
  - hint, success, and retry style;
  - reward usage;
  - asset priority.
- Defined shared emotion states.
- Mapped characters to worlds, skills, and interaction types.
- Added character asset naming and minimum asset set for Stage 17.

## Why It Matters

BrainUp now has a character system that can make future puzzles feel guided
without turning them into long tutorials. The characters also give Stage 18 a
clear `characterId` target for visual puzzle metadata.

## Next Step

Stage 17 should define the asset system: folders, naming conventions,
placeholder requirements, and production rules for puzzle, world, character,
reward, and animation assets.
