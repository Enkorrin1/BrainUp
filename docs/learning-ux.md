# UX и учебный цикл

## Главный принцип

Ребенок всегда должен понимать, что делать дальше. В интерфейсе должен быть один очевидный следующий шаг: начать текущий урок на карте, продолжить серию, повторить ошибки или открыть следующий узел.

## Основной цикл

1. Открыть приложение.
2. Увидеть Duolingo-style карту и текущую точку.
3. Начать короткий урок.
4. Решить серию задач.
5. Получить мгновенную обратную связь.
6. Завершить урок.
7. Получить XP, streak progress и обновление карты.
8. При ошибках увидеть предложение review.
9. Вернуться завтра ради daily goal и streak.

## Экран карты

Назначение: показать путь обучения и текущий прогресс.

Должен содержать:

- текущий активный урок;
- завершенные уроки;
- заблокированные будущие уроки;
- повторительные узлы;
- наградные узлы;
- прогресс по навыкам;
- streak;
- hearts;
- XP;
- daily goal;
- кнопку продолжения.

## Экран задания

Назначение: дать одну понятную задачу без лишнего шума.

Должен содержать:

- короткое условие;
- визуальную или текстовую задачу;
- варианты ответа или поле ввода;
- кнопку проверки;
- индикатор прогресса урока;
- hearts;
- hint action, если подсказка доступна.

## Обратная связь

После ответа ребенок должен сразу понять результат.

Если ответ верный:

- показать положительную реакцию;
- коротко подтвердить решение;
- перейти дальше без лишней задержки.

Если ответ неверный:

- не стыдить ребенка;
- списать heart только по правилам урока;
- показать, что именно не так;
- дать подсказку или объяснение;
- добавить задачу в mistake review.

## Экран результата

Назначение: завершить урок эмоционально и показать пользу.

Должен содержать:

- количество правильных ответов;
- accuracy;
- полученный XP;
- изменение streak;
- изменение hearts, если применимо;
- прогресс узла на карте;
- ошибки для повторения;
- кнопку next lesson;
- кнопку review mistakes, если есть ошибки.

## Повторение ошибок

Ошибки должны попадать в отдельную очередь. Повторение не должно выглядеть как наказание. Лучше позиционировать его как короткую тренировку, которая помогает вернуть hearts, укрепить навык или получить дополнительный XP.

Логика MVP:

- если задача решена неверно, добавить ее в mistake review;
- после урока предложить короткое повторение;
- считать ошибку закрытой после правильного повторного ответа;
- хранить skill tag ошибки для родительской аналитики.

## Мотивация

Использовать:

- streak;
- daily goal;
- XP;
- hearts;
- уровни узлов;
- достижения за регулярность;
- мягкие сообщения прогресса;
- коллекционные награды из текущего LogicLike-направления.

Избегать:

- агрессивного давления;
- слишком длинных уроков;
- сложной экономики;
- наказания за ошибки;
- рейтингов, которые демотивируют слабых учеников.
