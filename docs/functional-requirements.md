# Функциональные требования

## Профиль пользователя

Система должна:

- создавать профиль ученика;
- хранить имя или псевдоним;
- хранить возрастной диапазон;
- хранить прогресс по урокам;
- хранить серию дней;
- хранить опыт;
- хранить историю ошибок.

## Карта обучения

Система должна:

- показывать список навыков и уроков;
- отмечать завершенные уроки;
- блокировать уроки с невыполненными prerequisites;
- показывать текущий рекомендуемый урок;
- показывать прогресс освоения навыка.

## Урок

Система должна:

- запускать урок;
- показывать задания по одному;
- принимать ответ пользователя;
- проверять ответ;
- показывать обратную связь;
- переходить к следующему заданию;
- завершать урок;
- сохранять результат.

## Проверка ответов

Система должна:

- поддерживать разные типы ответов;
- сравнивать ответ с correctAnswer;
- учитывать частично правильные ответы там, где это предусмотрено;
- сохранять ошибочные ответы;
- показывать объяснение после ошибки.

## Прогресс

Система должна:

- начислять опыт;
- обновлять серию дней;
- обновлять прогресс урока;
- обновлять прогресс навыка;
- открывать следующие уроки;
- формировать очередь повторения.

## Повторение

Система должна:

- добавлять ошибочные задания в повторение;
- предлагать повторение после урока;
- считать задание повторенным после правильного ответа;
- показывать количество оставшихся ошибок.

## Родительский обзор

Для MVP достаточно простого экрана или блока прогресса.

Система должна показывать:

- количество завершенных уроков;
- активные дни;
- сильные навыки;
- слабые навыки;
- среднюю точность.
