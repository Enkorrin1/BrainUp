# Поэтапный план прототипа приложения

## Цель прототипа

Собрать первый рабочий Flutter-прототип BrainUp на базе технического опыта `D:\Project\LogicLike`: сохранить подход к логическим задачам и часть доменной основы, но сделать новый оригинальный визуальный дизайн и заменить главный учебный сценарий на Duolingo-style карту с короткими уроками, XP, streak, hearts и повторением ошибок.

Прототип считается успешным, если ребенок может:

1. открыть приложение;
2. увидеть карту обучения;
3. нажать текущий урок;
4. пройти 3-5 задач;
5. получить feedback;
6. увидеть результат урока;
7. увидеть обновленный прогресс на карте.

## Этап 1. Подготовка ветки и перенос основы

### Цель

Подготовить проектную основу, не ломая текущий `LogicLike`.

### Действия

- Создать отдельную рабочую ветку или копию проекта под BrainUp.
- Взять за основу папку `mobile` из `D:\Project\LogicLike`.
- Сохранить структуру:
  - `lib/src/app`;
  - `lib/src/data`;
  - `lib/src/domain`;
  - `lib/src/features`;
  - `lib/src/l10n`;
  - `lib/src/theme`;
  - `test`.
- Переименовать app-level сущности только там, где это не ломает сборку.
- Оставить Android/iOS runner совместимыми.

### Результат

Flutter-проект запускается, проходит базовые команды:

```powershell
cd mobile
flutter pub get
flutter analyze
flutter test
```

## Этап 2. Зафиксировать Duolingo-style доменную модель

### Цель

Добавить слой карты обучения поверх существующих `Lesson`, `LessonStep`, `PuzzleDefinition` и `ChildProfile`.

### Новые или обновленные модели

В `mobile/lib/src/domain`:

- `LearningPath`;
- `PathUnit`;
- `PathNode`;
- `PathNodeState`;
- `LessonResult`;
- `MistakeReviewItem`;
- `DailyGoal`;
- `HeartState`;
- `StreakState`.

### Что переиспользовать

Из текущего `LogicLike`:

- `Lesson`;
- `LessonStep`;
- `PuzzleDefinition`;
- `PuzzleType`;
- `SkillTag`;
- `ChildProfile`;
- `PracticeSession`;
- `FamilyProfile`.

### Результат

Есть тестируемая модель, которая умеет:

- определить текущий узел;
- определить locked/current/completed состояние;
- открыть следующий узел после завершения урока;
- сохранить XP, streak, hearts и ошибки.

## Этап 3. Главный экран-карта

### Цель

Заменить hub-first опыт на карту как первый экран после onboarding и сделать ее визуально самостоятельной, а не похожей на текущий экран `LogicLike`.

### Экран

Создать или переработать:

```text
mobile/lib/src/features/path/path_screen.dart
```

### Должно быть на экране

- Верхняя панель: hearts, streak, XP.
- Вертикальная или островная карта уроков в новом оригинальном стиле.
- Один clearly highlighted текущий урок.
- Завершенные уроки.
- Заблокированные уроки.
- Кнопка старта текущего урока.
- Вход в родительский раздел вторичным действием.

### Дизайн-ограничение

Нельзя копировать текущие композиции, карточки, персонажей, цветовую систему и reward-экран из `LogicLike`. Прототип должен использовать собственный визуальный язык.

### Результат

После onboarding пользователь попадает не в learning hub, а на карту обучения.

## Этап 4. Урок из 3-5 задач

### Цель

Собрать короткий урок в стиле Duolingo, но с текущими LogicLike-пазлами.

### Использовать

Текущую основу:

```text
mobile/lib/src/features/lesson/lesson_screen.dart
mobile/lib/src/domain/learning_foundation.dart
mobile/lib/src/domain/daily_challenge.dart
```

### Изменить

- Урок запускается из `PathNode`.
- Header урока показывает progress и hearts.
- Ошибка списывает heart по простому правилу MVP.
- Неверный ответ добавляется в mistake review.
- Подсказка сохраняется в статистику урока.
- В конце формируется `LessonResult`.

### Результат

Ребенок может пройти один урок от начала до конца.

## Этап 5. Экран результата урока

### Цель

Сделать reward moment после урока.

### Экран должен показать

- XP за урок.
- Accuracy.
- Streak update.
- Hearts state.
- Количество ошибок.
- Прогресс следующего узла.
- Primary action: продолжить.
- Secondary action: повторить ошибки, если они есть.

### Результат

После урока пользователь понимает, что получил, что открылось дальше и что делать следующим действием.

## Этап 6. Mistake Review MVP

### Цель

Добавить короткое повторение ошибочных задач.

### Правила MVP

- Неверная задача попадает в очередь review.
- Review запускается после урока или отдельным узлом карты.
- Правильный повторный ответ закрывает ошибку.
- Review может давать небольшой XP bonus.

### Результат

Ошибки становятся частью обучения, а не просто статистикой.

## Этап 7. Локальное сохранение прогресса

### Цель

Сохранить все важные состояния через текущий storage-подход.

### Сохранять

- completed path nodes;
- completed lessons;
- XP;
- hearts;
- current streak;
- best streak;
- daily goal state;
- mistake review queue;
- practice sessions.

### Использовать

```text
mobile/lib/src/data/family_profile_store.dart
```

### Результат

После перезапуска приложения прогресс не теряется.

## Этап 8. Локализация EN/RU

### Цель

Все новые строки сразу вести через Flutter l10n.

### Обновить

```text
mobile/lib/l10n/app_en.arb
mobile/lib/l10n/app_ru.arb
mobile/lib/src/l10n/generated/
```

### Покрыть строками

- карта;
- состояния уроков;
- hearts;
- streak;
- XP;
- результат урока;
- mistake review;
- empty/error states.

### Результат

Новые экраны не содержат hard-coded user-visible copy.

## Этап 9. Тесты прототипа

### Цель

Проверить не UI-красоту, а рабочую механику.

### Unit tests

Покрыть:

- расчет состояния узлов карты;
- открытие следующего урока;
- начисление XP;
- обновление streak;
- списание hearts;
- добавление ошибки в review;
- закрытие ошибки после повторения.

### Widget tests

Покрыть:

- открытие текущего урока с карты;
- прохождение урока;
- экран результата;
- появление review action после ошибки.

### Команды

```powershell
cd mobile
flutter analyze
flutter test
```

## Этап 10. Android smoke check

### Цель

Проверить прототип на реальном UI, а не только тестами.

### Проверить на emulator

- запуск приложения;
- onboarding;
- карта;
- старт урока;
- правильный ответ;
- неправильный ответ;
- завершение урока;
- возврат на карту;
- сохранение после перезапуска.

### Результат

Есть скриншоты и короткий список найденных проблем.

## Приоритет реализации

1. Domain model для карты и результата урока.
2. Path screen как новый главный экран.
3. Lesson launch from path.
4. Lesson result + progress update.
5. Local persistence.
6. Mistake review.
7. L10n.
8. Tests.
9. Android smoke.

## Что можно временно упростить

- Оставить текущие визуальные пазлы без полной переработки.
- Не делать сложную анимацию карты.
- Не делать полноценные daily quests.
- Не делать backend.
- Не делать оплату.
- Не делать AI.
- Не делать учительский кабинет.

## Что нельзя упрощать

- Главный экран должен быть картой, а не каталогом.
- Дизайн должен быть уникальным, а не прямой копией текущего `LogicLike`.
- Урок должен проходиться полностью.
- Прогресс должен сохраняться.
- Ошибки должны попадать в review.
- Все новые строки должны идти через l10n.
- Android/iOS совместимость нельзя ломать.

## Первый прототипный milestone

Минимальный milestone:

- 1 карта;
- 1 unit;
- 6 path nodes;
- 3 рабочих урока;
- 3-5 задач в уроке;
- XP;
- streak;
- hearts;
- lesson result;
- локальное сохранение;
- базовые тесты.

После этого можно оценивать UX и решать, насколько сильно визуально уходить от текущего LogicLike hub в сторону Duolingo-style learning path.
